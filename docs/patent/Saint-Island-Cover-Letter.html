OpenRNG Patent Family Strategy Briefing & Filing Instructions

一、第一案現狀 Current Status of Patent #1

第一案「遊戲隨機數生成方法與系統」（Method and System for Gaming Random Number Generation）已核准。該案保護的核心技術為：

  • Merkle-tree-based 批次隨機數生成
  • 區塊鏈錨定（Polygon 智能合約）
  • 每個隨機數值附帶 Merkle proof，可獨立驗證

第一案建立了整個專利家族的基礎，但它只解決了一半的問題：

第一案證明的是：隨機數值在承諾（commitment）之後沒有被竄改。

第一案沒有證明的是：隨機數值在承諾之前不能被預覽。

Patent #1: Merkle proves values were not changed after commitment.
Patent #1 does NOT prove values were not known before commitment.

這個缺口就是第二案要補的洞，也是整個專利家族從「單一專利」升級為「平台級 IP」的起點。

二、策略升級：從 OpenRNG 到 Computational Trust 從 OpenRNG 到計算信任

在第一案核准後，我們重新審視了整個技術的定位，做出了一個重要的策略轉向：

     │

核心轉變：我們不再將每件專利視為獨立的技術發明，而是將它們定位為統一架構（Computational Trust Infrastructure）中各自負責一層的組件。就像 Apple 的晶片家族（A 系列、M 系列、S 系列）不是獨立產品，而是統一平台的不同層級。

Computational Trust（計算信任）的定義：

獨立驗證計算過程——其輸入、執行和輸出——正確、公平且未被操縱地發生的能力，無需依賴操作者的聲明。

The ability to independently verify that a computational process — its inputs, execution, and outputs — occurred correctly, fairly, and without manipulation, without relying on the operator's assertion.

這不是安全（security）、不是隱私（privacy）、也不是合規（compliance）。它是一個新的基礎設施類別——如同 TLS 之於傳輸安全、OAuth 之於授權、JWT 之於身份。

三、為什麼第二案至關重要且必須盡快申請

Patent #2：VDF-Protected Batch RNG / Anti-Preview

Method and System for Verifiable Delay Function-Protected Batch Random Number Generation with Anti-Preview Guarantees

3.1 它解決什麼問題

第一案使用 Merkle tree 確保承諾後的完整性（post-commitment integrity）。但存在一個「內部預覽攻擊」（insider preview attack）：

  1. 系統操作者生成多個候選批次
  2. 在錨定 Merkle root 之前，預覽所有批次的數值
  3. 選擇性地錨定對操作者有利的批次
  4. 所有 Merkle 驗證檢查都通過——因為數值從未被修改，只是被選擇

第二案引入 VDF（可驗證延遲函數），創造一個時間屏障：在 VDF 計算完成之前（T 秒），沒有任何人——包括系統操作者——能夠知道批次中的任何數值。

第一案 + 第二案的組合提供「雙重時間完整性」（Dual Temporal Integrity）：

• 承諾前：數值不可知（VDF 保證）

• 承諾後：數值不可改（Merkle 保證）

Before commitment: values UNKNOWABLE (VDF guarantee)
After commitment: values UNALTERABLE (Merkle guarantee)

3.2 為什麼必須盡快

VDF 相關的 prior art 正在快速增加：

  • Boneh et al. (2018) 定義了 VDF 原語
  • Tezos (2022) 在共識層整合了 VDF
  • StarkWare VeeDo 實作了 VDF + 鏈上驗證
  • Ethereum 研究社群持續提出 VDF 應用
  • Chainlink 專利（US12401532B2）的 disclosure 中提到了 VDF

我們的 novel combination 尚未被任何 prior art 涵蓋（VDF + batch generation + Merkle tree + async anchoring + shared pool + drand integration + multi-worker architecture），但每多等一天，prior art 就會增加，新穎性的窗口就會縮小。

四、為什麼第三案同樣重要

Patent #3：Verifiable Entropy Objects (VEO)

Method and System for Generating, Scoring, Signing, Anchoring, and Verifying Verifiable Trust Artifacts for Non-Deterministic Computation

4.1 它解決什麼問題

第一案和第二案解決了「如何生成可信的隨機數」。第三案解決的是更大的問題：如何將信任封裝為一個可攜帶、可驗證的數位物件。

VEO 是一個結構化的信任成品（trust artifact），攜帶：

  • 熵載荷（entropy payload）
  • 來源出處記錄（source provenance）
  • 六維度品質評分（ECS, 0-1000）
  • 密碼學簽名（provider signature）
  • 可選的區塊鏈錨定（blockchain anchor）
  • 血統參考（lineage graph）

VEO 之於不確定性，如同 PDF 之於文件、JWT 之於身份。任何第三方都可以在離線、無網路的環境下，僅使用 VEO 物件和供應商的公鑰，獨立驗證整個信任斷言。

家族設計原則：

  • 無重疊：每件專利主張一個獨特的層，邊界在架構上定義且技術上強制執行
  • 交叉引用：每件專利將其他專利引用為相關申請，建立家族關係

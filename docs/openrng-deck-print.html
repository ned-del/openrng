The Problem / 問題

現有方案要麼僅限鏈上，要麼只返回簽名數字 — 沒有人提供完整的可驗證熵物件。

The Gap

On-chain VRFs (Chainlink, Pyth, Supra) require Solidity. Off-chain options like drand and RANDOM.ORG provide raw numbers or signed responses — but no provenance scoring, no blockchain anchoring, no replay verification.

Nobody packages randomness as a verifiable object. Existing off-chain solutions return a number with a signature. OpenRNG returns a proof-carrying entropy object with source provenance, confidence scoring, and chain-of-custody.

AI agents making autonomous decisions need more than a signed number — they need auditable, scored, anchored entropy.

鏈上 VRF 需要 Solidity。drand 和 RANDOM.ORG 提供原始數字或簽名回應，但沒有來源追蹤、信心評分、區塊鏈錨定或重播驗證。
沒有人將隨機性包裝成可驗證物件。

How It Works / 運作原理

多源熵收集 → VDF 時間鎖 → 密碼學簽名 → 區塊鏈錨定
Sources (drand · BTC · Polygon) → Verify → Score (ECS) → Sign (EIP-191) → VDF (Wesolowski) → Anchor (Polygon) → VEO-1
